# 操作备忘录

## 如何上传代码
```bash
# 1. 将修改后的文件添加到暂存区
git add README.md
# 2. 提交更改并添加描述信息
git commit -m "first commit"
# 3. 将本地代码推送到远程 GitHub 仓库的 main 分支。orgin就是git代码时候的地址。
git push origin main
```
命令动作
git add	工作区 → 暂存区	
git commit	暂存区 → 本地仓库
git push	本地仓库 → 远程仓库	

## 如何回滚
```bash
 git log --oneline --graph --all
# 软回滚 (推荐)：只撤销提交，保留代码
# 如果你觉得：“代码写得没错，但我提交得太仓促了，或者备注写错了。” HEAD~1 表示回到上一个版本
git reset --soft HEAD~1
# 硬回滚 (慎用)：彻底毁灭，回到过去
git reset --hard 52027d7

# 当你执行完上面的 reset 后，如果你直接执行 git push，GitHub 会报错（因为它不理解你为什么变旧了）。这时候你需要**“强行覆盖”**：
git push -f origin main
```
